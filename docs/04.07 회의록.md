        - 파일/퍼미션: Config.bm25_index_path 경로의 쓰기/읽기 권한과 .gitignore 처리(인덱스 파일이 커밋되지 않도록).
        - 동시성/성능: 다중 스레드/프로세스에서 사용 시 로드/세이브 시락(lock) 또는 프로세스별 인스턴스 권장.
        - 테스트: 단위 테스트(토큰화, 빌드, 질의 결과 포맷) 추가.
        
        권장 통합 흐름
        
        1. Config 검증: 파이프라인 시작 시 Config가 bm25 관련 값과 경로를 갖는지 확인.
        2. 인스턴스 생성: engine = BM25Engine(config)
            - 저장된 인덱스가 있으면 load_index()로 로드(이미 코드에 있음).
        3. 인덱스 업데이트: 새로운 chunks가 생기면 engine.build_index(chunks) 호출.
        4. 질의 호출: raw = engine.query(query_text, n_results=k) → 어댑터에서 평탄화(normalize)하여 RetrieverTemplate 규격 반환.
        5. 병렬/동시 접근 제어: build_index/save_index 시 임계 구역 설정.

        ```jsx
        from __future__ import annotations

        ```
