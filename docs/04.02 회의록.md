
## Task 검증

- 전체 5개 Task
    - 각 태스크 별 장점과 리스크 검증
        - 태스크 1개 선정 목표로 논의 진행
    - 검증 기준
        - 할당 된 시간 내에 가능한가?
        - 할 역량이 되는가?
        - 비슷한 프로젝트를 해본 경험이 있는가?
        - 양질의 데이터 셋을 확보할 수 있는가?

## Task 선정

- 기존 논문에서 사용한 방법론 - 모델, 자료구조, 프롬프팅 기법 정리
- 그래프 자료구조 적용 방안 고민 - 어떤 엔티티로 구성할 것인가, 프레임워크는?
- 연구계획서 초안 작성 - Task 세분화해서 적어보기 (모델 학습은 필요 X)
